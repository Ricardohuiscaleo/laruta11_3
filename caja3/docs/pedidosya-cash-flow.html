1. Resumen del Feature

Cuando un pedido de PedidosYA se paga en efectivo (el rider paga en el local), el cajero necesita:

  • Registrar que el pago es en efectivo (no online)
  • Cobrar al rider el precio PedidosYA (más alto que el precio de caja)
  • Calcular el vuelto si el rider paga con un billete grande
  • Registrar automáticamente el ingreso en caja

Antes: El cajero registraba manualmente un ingreso a caja por el excedente. Proceso propenso a errores.

Ahora: Todo automatizado — modal de selección Online/Efectivo, precios PYA automáticos, cash modal con vuelto, registro en caja automático.

2. Flujo Completo Paso a Paso

Fase 1: Creación del Pedido

1
Cajero arma el pedido

Agrega productos al carrito desde el menú de caja3. Los precios mostrados son los de caja.

2
Presiona "PedidosYA"

Se abre el modal de selección con dos opciones: 🌐 Online y 💵 Efectivo.

3
Elige "Efectivo"

Los precios del carrito cambian a precios PedidosYA (naranja, con precio caja tachado). El subtotal y total se actualizan. Aparece badge "🛵 Precio PedidosYA Efectivo aplicado".

4
Confirma el pedido

La orden se crea con payment_method: 'pedidosya_cash' y el monto PYA como total. Se redirige a la página de pedido pendiente.

Fase 2: Cobro al Rider

5
Pedido aparece en MiniComandas

Se muestra con etiqueta "PedidosYA Efectivo" y ícono 💵. Botón "CONFIRMAR PAGO" visible.

6
Cajero presiona "CONFIRMAR PAGO"

Se abre el Cash Modal mostrando el total PYA a cobrar. Botones rápidos: Monto Exacto, $5.000, $10.000, $20.000.

7
Ingresa monto del rider

Si el monto es mayor al total, muestra pantalla de confirmación con el vuelto. Si es exacto, confirma directo.

8
Confirma el pago

Se ejecuta: UPDATE payment_status → 'paid', INSERT en caja_movimientos, descuento de inventario. Todo en transacción.

5. APIs y Backend

get_menu_products.php

Agregado pedidosya_price al SELECT y al array de respuesta JSON. Permite al frontend conocer el precio PYA de cada producto.

confirm_transfer_payment.php

Extendida la condición de registro en caja:

// Antes
if ($order['payment_method'] === 'cash')

// Ahora
if (in_array($order['payment_method'], ['cash', 'pedidosya_cash']))

Para pedidosya_cash, el motivo en caja_movimientos es: "Venta PedidosYA Efectivo - Pedido #T11-XXXX"

get_sales_summary.php

Agregada categoría pedidosya_cash al array de resultados para que ArqueoApp pueda mostrar el desglose.

create_order.php

No requirió cambios — ya acepta cualquier string como payment_method.

6. Cambios en Base de Datos

ALTER TABLE — payment_method ENUM

ALTER TABLE tuu_orders MODIFY COLUMN payment_method
  ENUM('webpay','transfer','card','cash','pedidosya',
       'pedidosya_cash','rl6_credit','r11_credit')
  DEFAULT 'webpay';

Nueva columna — pedidosya_price

ALTER TABLE products
  ADD COLUMN pedidosya_price DECIMAL(10,2) DEFAULT NULL
  AFTER price;

7. Reportes y Trazabilidad

  • ArqueoApp: Tarjeta separada "PedidosYA Efectivo" (amber) con total y cantidad. Tarjeta "PedidosYA Online" para pagos online.
  • VentasDetalle: Badge "PYA Efectivo" en amarillo. Filtro disponible para ver solo pedidos pedidosya_cash.
  • Caja Movimientos: Cada pago PYA Efectivo genera un ingreso automático con motivo descriptivo y referencia a la orden.
